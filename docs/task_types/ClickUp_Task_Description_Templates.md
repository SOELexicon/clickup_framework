## Template Usage Guidelines

### Selecting the Right Template

1. **Coding Task**: Complex development work requiring detailed technical specifications
2. **Bug Fix Task**: Addressing defects with clear reproduction steps
3. **Feature Request**: New functionality or enhancements to existing features
4. **Research & Investigation**: Exploratory work to answer questions or evaluate options
5. **Documentation Task**: Creating or updating technical documentation
6. **Code Review**: Systematic review of code changes
7. **Deployment Task**: Releasing code to production or other environments
8. **Configuration Change**: Modifying system configuration without code changes
9. **Database Migration**: Schema or data changes to databases
10. **Performance Optimization**: Improving speed, efficiency, or resource usage

### Customization Tips

*   **Remove inapplicable sections**: Not all sections apply to every task
*   **Add project-specific fields**: Include custom fields relevant to your workflow
*   **Adjust detail level**: Scale detail based on task complexity
*   **Use consistent terminology**: Maintain consistency across your team
*   **Link related resources**: Always reference related tasks, docs, and repositories

### Best Practices

1. **Search before creating**: Always search for existing tasks before creating new ones
2. **Document thoroughly**: Comprehensive descriptions save time and prevent misunderstandings
3. **Include acceptance criteria**: Clear criteria prevent scope creep and ambiguity
4. **Add context**: Explain why, not just what
5. **Link dependencies**: Reference related tasks, blockers, and follow-up work
6. **Update status properly**: Follow natural workflow progressions
7. **Use memory storage**: Store task IDs for quick reference in future sessions
8. **Batch operations**: When making multiple changes, use batched API calls for efficiency
