## Quick Reference Guide

### Template Selection Matrix

| Task Characteristics | Recommended Template |
|---------------------|---------------------|
| New feature development | Coding Task |
| Fixing broken functionality | Bug Fix Task |
| Adding new capability | Feature Request |
| Investigating options | Research & Investigation |
| Writing docs | Documentation Task |
| Reviewing code | Code Review |
| Pushing to production | Deployment Task |
| Changing settings | Configuration Change |
| Altering database | Database Migration |
| Making things faster | Performance Optimization |

### Best Practices

1. **Always start with the template**: Choose the most appropriate template before creating a task
2. **Customize as needed**: Remove sections that don't apply; add project-specific fields
3. **Fill all placeholders**: Replace all `[bracketed placeholders]` with actual information
4. **Link related work**: Reference related tasks, docs, and repositories
5. **Define acceptance criteria**: Always include clear, testable acceptance criteria
6. **Document context**: Explain the "why" behind the work, not just the "what"

### Template Usage

1. Open the appropriate template file from the list above
2. Copy the entire template content
3. Create a new ClickUp task
4. Paste the template into the task description
5. Customize all placeholders marked with `[...]`
6. Remove sections that don't apply to your specific task
7. Add any additional project-specific information
