## Objective
Fix the `cum task_move` / `cum tmv --list` bug so moving a task to another list actually changes the task's home list in ClickUp instead of only printing a false success message.

## Context
- Current state: `clickup_framework/commands/task_commands.py` builds `updates['list'] = dest_list_id` and calls `client.update_task(task_id, **updates)`.
- Problem: On April 3, 2026 this path printed `✓ Moved: ...` for multiple tasks, but `client.get_task(task_id)['list']['id']` still returned the original list.
- Constraints or assumptions:
  - The ClickUp Multiple Lists add/remove endpoints are not a safe fallback here; they returned `403` due to feature-usage limits.
  - A direct v3 home-list move request worked: `PUT /api/v3/workspaces/{workspace_id}/tasks/{task_id}/home_list/{list_id}`.
  - Related closed feature task exists for move support: `86c79tz9v`.

## Steps to Reproduce
1. Run `python -m clickup_framework tmv 86c79qgm6 --list 901519249130 --force`.
2. Observe that the command prints a success message.
3. Fetch the task again with the API and inspect `task['list']['id']`.

**Expected Behavior:** The task's home list changes to the destination list and subsequent reads show the new list ID.

**Actual Behavior:** The command prints success, but the task remains in the original list unless the v3 `home_list` endpoint is called directly.

## Root Cause
- The command currently treats list moves like a generic task update.
- ClickUp does not move the task's home list via `update_task(..., list=...)` in this flow.
- The CLI therefore reports a successful move without validating that the task actually changed lists.

## Work To Be Done
1. Replace the broken `update_task(..., list=...)` move path with the correct ClickUp home-list move implementation.
2. Preserve parent-only moves and avoid breaking current `--parent` behavior.
3. Make success output reflect the real destination and fail loudly if the list did not change.
4. Add regression coverage for cross-list move behavior and the false-success case.
5. Update help text and documentation so examples match the fixed implementation.
