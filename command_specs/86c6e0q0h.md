# task_unassign - Remove Assignees from Tasks

**Task ID:** 86c6e0q0h
**Status:** ✅ Implemented
**Command:** `cum task_unassign` / `clickup task_unassign`

## Synopsis

```bash
cum task_unassign <task_id> <assignee_id> [assignee_id ...]
```

Remove one or more users from a task's assignee list.

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `task_id` | string | ✓ | Task ID or "current" from context |
| `assignee_ids` | string[] | ✓ | One or more user IDs to remove |

## Examples

### Remove Single User
```bash
cum task_unassign 86c6e0xyz 12345678
```

### Remove Multiple Users
```bash
cum task_unassign 86c6e0xyz 12345678 87654321
```

### Using Context
```bash
cum set_current task 86c6e0xyz
cum task_unassign current 12345678
```

## Output

### Success
```
✓ Removed 1 user(s) from task

Task: API Development
```

### Error
```
Error unassigning task: ClickUp API Error 404: Task not found
```

## Behavior

- Removes specified users from assignee list
- Other assignees remain unchanged
- No error if user wasn't assigned
- Task can have zero assignees after removal

## API Endpoint

`PUT /task/{task_id}` with `assignees.rem` parameter

## Notes

- Removing non-existent assignees is silently ignored
- Can remove all assignees (task becomes unassigned)
- Multiple users can be removed in one command
- Context support via `set_current task <id>`

## Related Commands

- [`task_assign`](#86c6e0q0g) - Add assignees
- [`task_update`](#86c6e0q0d) - Update other task fields
- `detail` - View current assignees

## Use Cases

### ✅ Good For
- Removing users from completed tasks
- Transferring work (unassign + assign)
- Cleaning up bulk assignments
- Removing users who left project

### Workflow Example
```bash
# Transfer task from user A to user B
cum task_unassign 86c6e0xyz 12345678
cum task_assign 86c6e0xyz 87654321

# Or clear all and reassign
cum task_unassign 86c6e0xyz 12345678 87654321 11223344
cum task_assign 86c6e0xyz 99887766
```

## Validation

- ✓ Task must exist
- ⚠️ User IDs are not validated (silently ignored if not assigned)

## Notes

- No confirmation prompt
- Cannot remove yourself from watchers (different API)
- Removing last assignee leaves task unassigned
- API handles notifications automatically
