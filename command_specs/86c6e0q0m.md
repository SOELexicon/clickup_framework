# task_set_priority - Set Task Priority

**Task ID:** 86c6e0q0m
**Status:** ✅ Implemented
**Command:** `cum task_set_priority` / `clickup task_set_priority`

## Synopsis

```bash
cum task_set_priority <task_id> <priority>
```

Set task priority using numeric (1-4) or name values.

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `task_id` | string | ✓ | Task ID or "current" from context |
| `priority` | int/string | ✓ | Priority: 1-4 or urgent/high/normal/low |

## Priority Levels

| Numeric | Name | Description |
|---------|------|-------------|
| 1 | urgent | Highest priority |
| 2 | high | High priority |
| 3 | normal | Normal/default priority |
| 4 | low | Lowest priority |

## Examples

### Numeric Priority
```bash
cum task_set_priority 86c6e0xyz 1
```

### Named Priority
```bash
cum task_set_priority 86c6e0xyz urgent
cum task_set_priority 86c6e0xyz high
cum task_set_priority 86c6e0xyz normal
cum task_set_priority 86c6e0xyz low
```

### Using Context
```bash
cum set_current task 86c6e0xyz
cum task_set_priority current urgent
```

## Output

### Success
```
✓ Priority updated

Task: API Development
New priority: 1
```

### Error
```
Error: Invalid priority 'critical'. Use 1-4 or urgent/high/normal/low
```

## Validation

- ✓ Accepts numeric: 1, 2, 3, 4
- ✓ Accepts names: urgent, high, normal, low (case-insensitive)
- ✓ Validates input before API call
- ✗ Rejects invalid values

## API Endpoint

`PUT /task/{task_id}` with `priority` parameter

## Notes

- Priority names are case-insensitive ("Urgent" = "urgent")
- Invalid values error before API call
- Context support via `set_current task <id>`
- No subtask validation (unlike task_set_status)

## Priority Display

In CLI output, priorities show as:
- **1/Urgent**: Bright red
- **2/High**: Orange/yellow
- **3/Normal**: White/default
- **4/Low**: Gray

## Use Cases

### ✅ Good For
- Quick priority updates
- Triaging tasks
- Escalating urgent issues
- Bulk priority changes via scripts

### Priority Workflow Example
```bash
# Escalate to urgent
cum task_set_priority 86c6e0xyz urgent

# Later, downgrade after completion
cum task_set_priority 86c6e0xyz low
```

## Clearing Priority

To remove priority (set to none):
```bash
# ClickUp API doesn't support null priority
# Use lowest priority instead
cum task_set_priority 86c6e0xyz 4
```

## Validation Details

### Valid Inputs
- `1`, `2`, `3`, `4` (integers)
- `"1"`, `"2"`, `"3"`, `"4"` (strings)
- `urgent`, `URGENT`, `Urgent` (case-insensitive)
- `high`, `normal`, `low` (case-insensitive)

### Invalid Inputs
- `0`, `5`, `-1` (out of range)
- `critical`, `medium` (not recognized)
- Empty string or null

### Error Message
```
Error: Invalid priority 'X'. Use 1-4 or urgent/high/normal/low
```
