# task_update - Update Existing Tasks

**Task ID:** 86c6e0q0d
**Status:** ✅ Implemented
**Command:** `cum task_update` / `clickup task_update`

## Synopsis

```bash
cum task_update <task_id> [OPTIONS]
```

Update one or more fields of an existing task.

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `task_id` | string | ✓ | Task ID or "current" from context |

## Options

| Option | Type | Description |
|--------|------|-------------|
| `--name TEXT` | string | New task name |
| `--description TEXT` | string | New task description |
| `--status TEXT` | string | New status (must exist in list) |
| `--priority INT` | 1-4 | New priority level |
| `--add-tags TAG [...]` | list | Tags to add |
| `--remove-tags TAG [...]` | list | Tags to remove |

## Examples

### Update Name
```bash
cum task_update 86c6e0xyz --name "Updated Task Name"
```

### Update Multiple Fields
```bash
cum task_update 86c6e0xyz \
  --name "Refactor Authentication" \
  --description "Complete auth system refactor" \
  --status "in development" \
  --priority 1
```

### Manage Tags
```bash
# Add tags
cum task_update 86c6e0xyz --add-tags "urgent" "reviewed"

# Remove tags
cum task_update 86c6e0xyz --remove-tags "needs-review"
```

### Using Context
```bash
cum set_current task 86c6e0xyz
cum task_update current --status "testing"
```

## Output

### Success
```
✓ Task updated: Refactor Authentication

Updated fields:
  name: Refactor Authentication
  description: Complete auth system refactor
  status: in development
  priority: 1
```

### Error
```
Error updating task: ClickUp API Error 404: Task not found
```

## Validation

- ✓ Task must exist
- ✓ Status must be valid for list (if provided)
- ✓ Priority must be 1-4 (if provided)
- ⚠️ No subtask validation (use `task_set_status` for that)

## API Endpoint

`PUT /task/{task_id}`

## Notes

- At least one option must be provided
- Tag operations fetch current task state first
- All updates are atomic (all or nothing)
- For validated status updates, use `task_set_status`
- Context support via `set_current task <id>`
