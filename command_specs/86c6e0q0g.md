# task_assign - Assign Users to Tasks

**Task ID:** 86c6e0q0g
**Status:** ✅ Implemented
**Command:** `cum task_assign` / `clickup task_assign`

## Synopsis

```bash
cum task_assign <task_id> <assignee_id> [assignee_id ...]
```

Assign one or more users to a task.

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `task_id` | string | ✓ | Task ID or "current" from context |
| `assignee_ids` | string[] | ✓ | One or more user IDs to assign |

## Examples

### Assign Single User
```bash
cum task_assign 86c6e0xyz 12345678
```

### Assign Multiple Users
```bash
cum task_assign 86c6e0xyz 12345678 87654321 11223344
```

### Using Context
```bash
cum set_current task 86c6e0xyz
cum task_assign current 12345678
```

## Output

### Success
```
✓ Assigned 1 user(s) to task

Task: API Development
Assignees: 12345678
```

### Error
```
Error assigning task: ClickUp API Error 400: User not found
```

## Behavior

- Adds new assignees to existing assignees (doesn't replace)
- Duplicate assignees are automatically handled by API
- Maintains existing assignees
- Updates task assignee list

## API Endpoint

`PUT /task/{task_id}` with `assignees.add` parameter

## Notes

- User IDs must be valid workspace members
- Assignees accumulate (use task_unassign to remove)
- Multiple assignees can be added in one command
- Does not validate if user has task permissions
- Context support via `set_current task <id>`

## Related Commands

## Validation

- ✓ Task must exist
- ✓ User IDs must be valid
- ✓ Users must have workspace access
- ⚠️ No permission level validation

## Use Cases

### ✅ Good For
- Adding collaborators to tasks
- Assigning work to team members
- Adding multiple assignees at once
- Bulk assignments via scripting

### ⚠️ Limitations
- Cannot change assignee permissions
- Cannot assign guests without workspace access
- No notification control (API handles)
