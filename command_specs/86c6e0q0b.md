# task_create - Create New Tasks

**Task ID:** 86c6e0q0b
**Status:** ✅ Implemented
**Command:** `cum task_create` / `clickup task_create`

## Synopsis

```bash
cum task_create <list_id> "<task_name>" [OPTIONS]
```

Create a new task in a specified list with comprehensive configuration options.

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `list_id` | string | ✓ | List ID or "current" from context |
| `task_name` | string | ✓ | Name/title of the task |

## Options

| Option | Type | Description |
|--------|------|-------------|
| `--description TEXT` | string | Detailed task description |
| `--status TEXT` | string | Initial status (must exist in list workflow) |
| `--priority INT` | 1-4 | Priority: 1=Urgent, 2=High, 3=Normal, 4=Low |
| `--tags TAG [...]` | list | One or more tags |
| `--assignees ID [...]` | list | User IDs to assign |
| `--parent ID` | string | Parent task ID (creates subtask) |

## Examples

### Basic Task
```bash
cum task_create 901517404274 "Fix login bug"
```

### Full Configuration
```bash
cum task_create 901517404274 "API Development" \
  --description "Create RESTful endpoints for user management" \
  --status "in development" \
  --priority 1 \
  --tags "backend" "api" "urgent" \
  --assignees 12345678 \
  --parent 86c6e0xyz
```

### Using Context
```bash
cum set_current list 901517404274
cum task_create current "New Feature"
```

## Output

### Success
```
✓ Task created: API Development

Task ID: 86c6e0xyz
URL: https://app.clickup.com/t/86c6e0xyz
```

### Error
```
Error creating task: ClickUp API Error 400: Status does not exist
```

## Validation

- ✓ List must exist
- ✓ Status must be valid for list (if provided)
- ✓ Priority must be 1-4 (if provided)
- ✓ Parent task must exist in same list (if provided)
- ✓ Assignees must have workspace access (if provided)

## API Endpoint

`POST /list/{list_id}/task`

## Notes

- Task name is required and cannot be empty
- All fields except name are optional
- Status defaults to list's default status
- Tags are created if they don't exist
- Context support via `set_current list <id>`
